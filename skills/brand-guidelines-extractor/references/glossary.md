# Brand Guidelines Extractor Glossary

## Overview

One of the inputs for downstream styling skills is a Brand Guidelines
document, produced by the Brand Guidelines Extractor. This glossary
explains how that document is structured: what each section contains,
what values to expect, and how to interpret them.

**Producing skill:** Brand Guidelines Extractor
**Current version:** v1.0
**Document type:** Brand Guidelines
**Purpose:** Documents a brand's colors, typography, and visual styling rules for consistent application across outputs.
**Consumers:** Any skill that applies brand styling to presentations, documents, artifacts, or other formatted output.

## Document Structure

| Section | Description |
|---|---|
| Overview | One paragraph identifying the brand, source URL, and purpose of the file. |
| Colors | Hex codes, plain-English names, and usage roles for all brand colors, split into Main Colors and Accent Colors. |
| Typography | Font families for headings and body text, with fallback fonts noted. |
| Smart Font Application | Rules for which font applies to which element type, including size thresholds and fallback behavior. |
| Text Styling | Color-on-background pairings for headings and body text, including inverted/dark-mode treatment. |
| Shape and Accent Colors | Rules for applying accent colors to non-text elements like borders, backgrounds, and decorative shapes. |
| Font Management | Practical notes on font sources, installation, and fallback chains. |
| Color Application | Color format details, CSS custom properties if available, and cross-system fidelity notes. |
| Confidence Notes | Present only when input was limited (e.g., screenshots only). Flags which values are approximate. |

## Section Hierarchy

When multiple sections provide guidance that applies to the same output:

1. Colors / Typography. These are facts. Apply them exactly.
2. Smart Font Application / Text Styling. Specific rules for element types. Follow these over general guidance.
3. Shape and Accent Colors. Directional guidance for non-text elements. Follow the cycling order if specified.
4. Overview. Context only. Does not override any specific section.
5. Confidence Notes. If present, treat flagged values as approximate and allow reasonable interpretation.

## Handling Missing or Unexpected Input

**A section says "unable to determine" or equivalent:**
Do not invent values. Fall back to safe defaults (black text, white
background, system sans-serif for headings, system serif for body).
Note to the user what was missing.

**An expected section is missing entirely:**
Treat the same as "unable to determine." Apply safe defaults for that
aspect of styling.

**The version marker is older than the current glossary version:**
Check the changelog. Apply current definitions, but account for
structural differences noted for that version.

**The document was not generated by this skill:**
The user may have written their own brand guidelines. Do not reject it.
Extract whatever color, font, and usage guidance is present, map it to
the fields you understand, and apply it. Fall back to safe defaults
where coverage is missing.

## Field Definitions

### Overview

**What it represents:** Context about the brand and where the data came from.
**What to expect:** One paragraph with brand name, URL, and a note about extraction method.
**How to apply:** Read for context. Do not extract styling values from this section.

### Colors: Main Colors

**What it represents:** The core palette used for backgrounds, text, and neutral elements.
**What to expect:** A list of 3-6 colors, each with a hex code, plain-English name (e.g., "dark charcoal"), and a role (e.g., "Primary text and dark backgrounds").
**How to apply:** Match hex codes exactly. Use the role descriptions to decide which color applies to which element. "Primary text" means default text color. "Light backgrounds" means the default page/canvas background.

### Colors: Accent Colors

**What it represents:** Non-neutral colors used for interactive elements, CTAs, decorative shapes, and visual emphasis.
**What to expect:** A list of 1-5 colors with hex codes, plain-English names, and usage notes. Colors are listed in priority order (first = primary accent).
**How to apply:** Use the first accent color for primary interactive elements (buttons, links, CTAs). Use secondary/tertiary accents for variety in non-text elements. If cycling is specified, follow the listed order.

### Typography

**What it represents:** The font families used for headings and body text.
**What to expect:** Two entries at minimum: one for headings, one for body text. Each includes the primary font family and a fallback. May include a note about whether fonts are custom or system-installed.
**How to apply:** Apply the heading font to all headings (H1-H6) and the body font to all other text. If the primary font is unavailable, use the listed fallback.

### Smart Font Application

**What it represents:** Detailed rules for font assignment by element type.
**What to expect:** A list of rules specifying font, element type, size threshold, and fallback behavior.
**How to apply:** Follow the rules as stated. If a size threshold is given (e.g., "24pt and larger"), use it to distinguish headings from body text in contexts where semantic heading tags are unavailable (like presentations).

### Text Styling

**What it represents:** Color-on-background pairings for text elements.
**What to expect:** Rules like "dark text on light backgrounds" and "light text on dark backgrounds" with specific hex values.
**How to apply:** Select text color based on the background color of the element. If a contrast rule is stated, follow it.

### Shape and Accent Colors

**What it represents:** How to apply accent colors to non-text elements.
**What to expect:** Rules for borders, backgrounds, buttons, and decorative shapes. May specify a cycling order for multiple accents.
**How to apply:** When styling non-text elements, use accent colors in the order listed. If a cycling pattern is described, rotate through the accents to maintain visual variety.

### Confidence Notes

**What it represents:** Flags indicating which values are approximate.
**What to expect:** Present only when the extraction used screenshots or partial data. Lists which categories of values (colors, fonts) are approximate and why.
**How to apply:** If precision matters for the downstream output, treat flagged values as estimates. If the downstream skill can tolerate slight variation (e.g., a presentation), apply them as-is.
